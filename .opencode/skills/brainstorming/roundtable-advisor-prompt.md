# 圆桌会议 Advisor Prompt 模板

当调度设计顾问子代理会话时使用此模板（三个顾问并行，每个使用相同模板）。

**目的：** 在主代理形成推荐方向后、写功能文档前，从三个独立视角评估设计方向

**注意：三个顾问必须并行调度，且每个顾问不能看到其他顾问的输出。**

```
设计顾问子代理会话：
  description: "圆桌会议：[GPT/Sonnet/Gemini] 视角"
  prompt: |
    你是设计顾问，正在独立评估以下设计方向。

    ## 问题 / 需求描述

    [完整粘贴用户的原始需求和上下文]

    ## 当前推荐的设计方向

    [完整粘贴主代理形成的当前设计立场和推荐方向，包括关键设计决策]

    ## 你的任务

    独立评估上述设计方向。不要参考任何其他顾问的分析。
    按规定格式输出你的立场。
```

## 主代理的聚合职责

收到三份顾问报告后，主代理需要：

1. **识别共识**：三方都认同的部分 → 在用户确认后纳入规格
2. **识别分歧**：找出有方向性分歧的点（不是细节措辞差异）
3. **整理分歧清单**，格式如下：

```
## 圆桌会议结果

### 共识
[三方都认同的设计判断]

### 分歧点 1：[分歧标题]
- GPT 视角：[立场摘要]
- Sonnet 视角：[立场摘要]
- Gemini 视角：[立场摘要]
→ 请裁决：[给用户的选择项或开放问题]

### 分歧点 2：...
```

4. 将分歧清单提交用户裁决
5. 用户裁决结果作为约束，进入写功能文档阶段

## 跳过条件

对于以下情形，主代理可跳过圆桌会议直接写规格（跳过时需说明原因）：

- 极简任务（单文件改动、配置调整、文档修改等）
- 设计方向已经非常明确且无可争议的技术决策
- 用户明确要求跳过
