# Frontend Design

这个 skill 用来指导你在本项目中创建有辨识度、可上线、设计质量高的前端界面，避免 generic AI aesthetics。

你必须输出真实可运行的代码，并且对审美细节、信息密度和整体气质保持高标准。

## 设计思考

写代码前，先理解上下文，并明确选择一个**清晰的审美方向**：

- **目的**：这个界面解决什么问题？谁来使用？
- **语气**：选择明确方向，例如极简克制、编辑式、工业感、研究工具感、原始粗粝、自然有机、几何秩序、玩具感、复古未来感等。不要停在模糊的“现代、简洁、好看”。
- **约束**：框架、性能、可访问性、已有设计语言、现有组件结构
- **记忆点**：这个界面最让人记住的是什么？必须至少有一个可感知的视觉判断，而不是“整体还行”

**关键要求：** 选一个明确的概念方向，并把它精确执行出来。克制的极简和强烈的风格化都可以，关键是有意图，而不是像模板。

## 本项目的额外约束

- 本项目是研究工具，不是 AI 玩具展示页
- 默认避免“紫色渐变 + 大圆角 + 毛玻璃 + 漂浮卡片”的 AI 产品通用模板
- 默认避免 Element 后台味、Electron 套壳味、通用 SaaS 仪表盘味
- 优先用排版、边界、密度、层级、留白和局部强调来建立辨识度
- 当前样式栈是 `Vue + UnoCSS`

## 本项目当前优先方向

- 当前已经确定的默认方向是“仿书页 / 档案页 / 研究手稿”，不是“研究控制台 / 科技工作台”
- 背景优先接近纸面、册页、档案纸，而不是纯白后台或深色面板
- 文字优先接近墨色、棕黑、灰褐等阅读型色系，而不是高饱和品牌蓝紫
- 标题可以用有气质的衬线字，正文必须首先耐读
- 分隔更像版心、栏线、编目、注记，而不是一堆卡片阴影

## 可读性规则

- 切到中文衬线或书卷气字体后，要主动检查字号是否仍然清晰；不要沿用无衬线模板里的偏小字号
- 长时间阅读的正文、列表项、输入框、按钮正文，默认至少 `text-base`
- `text-sm` 用于辅助说明、次级说明、时间戳、过滤提示
- `text-xs` 只用于非常短的小标签或眉题，不要让它承担主要信息
- 正文和说明文字要同步提高 `line-height`，避免字面挤在一起

## 前端审美指南

重点关注：

### 字体

- 字体要有气质，不要无脑退回 `Arial`、`Roboto`、`Inter`
- 正文和标题可以形成明确分工：标题更有个性，正文更耐读
- 如果无法安全引入外部字体，至少通过字重、字距、字号节奏做出差异，而不是全部一把梭

### 色彩与主题

- 先确定主导色，再决定强调色，不要平均分配颜色权重
- 用少量高质量颜色，胜过一堆浅色层叠
- 研究工具默认更适合克制、冷静、可信的色彩体系，而不是高饱和情绪型品牌色

### 动效

- 动效服务于节奏和反馈，不是为了"显得高级"
- 优先使用少量但高质量的关键动效：进入、切换、hover、focus
- 如果页面整体很克制，动效也要克制；不要出现视觉语言和动画语言互相打架

#### 本项目动画约定

- **出现/入场**：`ease-out`，时长 150–200ms；优先 `opacity + translate-y` 组合
- **退出/离场**：`ease-in`，时长 100–150ms
- **交互反馈（hover/active）**：`ease-in-out`，时长 100–150ms；`hover:scale-[1.02]` / `active:scale-95`
- **小惊喜（偶发）**：scale 弹入 `scale-95 → scale-100`，或 translate-y 上滑，传达书页翻阅感
- **实现方式**：使用 Vue `<Transition>` / `<TransitionGroup>` 配合 UnoCSS `transition-*` utilities，不手写 JS 动画
- **禁止**：`animation-duration > 400ms` 的纯装饰动画、旋转 360°、过强 bounce、纯装饰闪烁
- **scale 幅度**：不超过 5%，避免破坏书页的静态严肃感
- **Toast 动画**：从顶部 `translate-y(-100%) scale-95` 入场，`opacity-0 translate-y(-8px)` 离场

### 空间与构图

- 可以使用不完全对称、轻微错位、边界破格，但必须有秩序
- 要么留白充足，要么密度明确，不要中间态的犹豫布局
- 让信息结构先成立，再做装饰

### 背景与细节

- 不要默认纯白卡片堆叠就是“完成设计”
- 可以用表面层次、细腻边框、纹理、微弱背景变化建立气氛
- 所有细节必须服从整体概念，而不是东拼西凑效果词

## 明确禁止

- `Inter`、`Space Grotesk`、默认系统无衬线作为所有方案的机械默认值
- 白底紫渐变、蓝紫发光、毛玻璃、超大圆角、悬浮卡片瀑布流
- 典型“AI 生成落地页”结构：英雄区大标题 + 三个 feature 卡片 + 紫色 CTA
- 为了看起来“现代”而堆积阴影、描边、渐变和动画
- 每次都生成同一种审美

## 实现要求

实现出的代码必须：

- 真实可运行
- 达到生产可用质量
- 有明确一致的审美立场
- 细节精致，而不是靠模板堆砌
- 与现有技术栈兼容（这里默认是 Vue + UnoCSS）

## 与本项目协作时的规则

- 先看已有页面的密度、布局和结构，再决定怎么升级它
- 如果在现有页面上做改动，优先延续其信息结构，而不是整个推翻成 landing page
- 不要顺手抽象一个庞大的 design system
- 除非用户明确要求，否则不要为了“做得更漂亮”去引入重量级 UI 库

## 快速判断

| 问题 | 要求 |
|------|------|
| 这个设计有没有明确气质？ | 必须有 |
| 会不会一眼看出像 AI 模板？ | 不能 |
| 会不会像 Element 后台壳？ | 不能 |
| 信息结构是否先于装饰成立？ | 必须 |
| 视觉细节是否服从整体概念？ | 必须 |

## 复杂度匹配

- 风格强烈的设计，需要对应强烈且一致的实现细节
- 克制型设计，不代表偷懒；它要求更准确的间距、排版和边界处理
- 优雅不是“少写代码”，而是每一个选择都足够准确

## 常见错误

**把“高设计感”理解成堆效果**

- 问题：结果是廉价炫技
- 修复：先决定概念，再决定哪些效果真的服务于概念

**把“AI 友好”理解成“用 AI 常见模板”**

- 问题：产出会迅速撞脸
- 修复：让 AI 服从项目审美，而不是向 AI 常见审美靠拢

**中文衬线字体换上了，但字号还停留在模板默认值**

- 问题：会显得发灰、发紧、看不清
- 修复：主动提高基础字号和行高，不要让长期可见信息停留在 `text-xs` / 偏小的 `text-sm`

**改一个业务页却做成 marketing 页面**

- 问题：视觉语言和产品性质错位
- 修复：先尊重界面的任务属性，再决定风格表达强度

记住：你有能力做出真正有审美判断的界面。不要自动收敛到最常见、最安全、最像模板的答案。
