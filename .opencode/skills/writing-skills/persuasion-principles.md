# Skill 设计的说服原则

## 概述

LLM 对与人类相同的说服原则有反应。理解这种心理学有助于你设计更有效的 skills——不是为了操控，而是为了确保即使在压力下也能遵循关键实践。

**研究基础：** Meincke 等人（2025）使用 N=28,000 次 AI 对话测试了 7 种说服原则。说服技术使合规率提高了一倍以上（33% → 72%，p < .001）。

## 七项原则

### 1. 权威
**定义：** 服从专业知识、资质或官方来源。

**在 skills 中的作用：**
- 命令性语言："你必须"、"绝不"、"始终"
- 不可协商的框架："无例外"
- 消除决策疲劳和 rationalization

**使用时机：**
- 强制执行纪律的 skills（TDD、验证要求）
- 安全关键实践
- 已建立的 best practices

**示例：**
```markdown
✅ 先写代码再写测试？删除它。重新开始。无例外。
❌ 在可行的情况下考虑先写测试。
```

### 2. 承诺
**定义：** 与过去的行动、陈述或公开声明保持一致。

**在 skills 中的作用：**
- 要求公告："宣布 skill 使用情况"
- 强制明确选择："选择 A、B 或 C"
- 使用跟踪：用 TodoWrite 建立检查清单

**使用时机：**
- 确保 skills 实际被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到一个 skill 时，你必须宣布："我正在使用 [Skill 名称]"
❌ 考虑让你的 partner 知道你正在使用哪个 skill。
```

### 3. 稀缺性
**定义：** 来自时间限制或有限可用性的紧迫感。

**在 skills 中的作用：**
- 有时间限制的要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**使用时机：**
- 立即验证要求
- 时间敏感的 workflow
- 防止"我稍后再做"

**示例：**
```markdown
✅ 完成任务后，在继续之前立即requesting-code-review。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会证明
**定义：** 遵循他人的行为或被认为正常的事情。

**在 skills 中的作用：**
- 普遍模式："每次"、"始终"
- 失败模式："没有 Y 的 X = 失败"
- 建立规范

**使用时机：**
- 记录通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的检查清单 = 步骤被跳过。每次都这样。
❌ 一些人发现 TodoWrite 对检查清单很有帮助。
```

### 5. 统一
**定义：** 共同身份、"我们感"、群体归属感。

**在 skills 中的作用：**
- 协作语言："我们的代码库"、"我们是同事"
- 共同目标："我们都想要质量"

**使用时机：**
- 协作 workflow
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是共同工作的同事。我需要你诚实的技术判断。
❌ 如果我错了，你可能应该告诉我。
```

### 6. 互惠
**定义：** 回报所获得利益的义务。

**如何起作用：**
- 谨慎使用——可能感觉像操控
- 在 skills 中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜爱
**定义：** 偏好与我们喜欢的人合作。

**如何起作用：**
- **不要用于合规**
- 与诚实反馈文化相冲突
- 创造奉承行为

**何时避免：**
- 在纪律执行中始终避免

## 按 Skill 类型组合原则

| Skill 类型 | 使用 | 避免 |
|------------|-----|-------|
| 强制纪律 | 权威 + 承诺 + 社会证明 | 喜爱、互惠 |
| 指导/技术 | 适度权威 + 统一 | 强权威 |
| 协作 | 统一 + 承诺 | 权威、喜爱 |
| 参考 | 仅清晰度 | 所有说服 |

## 为什么有效：心理学原理

**明确的规则减少 rationalization：**
- "你必须"消除决策疲劳
- 绝对语言消除"这是例外吗？"的问题
- 明确的反 rationalization 反制措施关闭特定漏洞

**实施意图创造自动行为：**
- 清晰的触发器 + 必需的行动 = 自动执行
- "当 X 时，做 Y"比"通常做 Y"更有效
- 降低合规的认知负担

**LLM 是类人的：**
- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规出现
- 承诺序列（陈述 → 行动）被频繁建模
- 社会证明模式（每个人都做 X）建立规范

## 道德使用

**合法的：**
- 确保遵循关键实践
- 创建有效的文档
- 防止可预见的失败

**不合法的：**
- 为个人利益操控
- 制造虚假紧迫感
- 基于内疚的合规

**测试：** 如果用户完全理解这种技术，它会符合他们的真实利益吗？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七项说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 使用 N=28,000 次 LLM 对话测试了 7 种原则
- 通过说服技术将合规率从 33% 提高到 72%
- 权威、承诺、稀缺性最有效
- 验证了 LLM 行为的类人模型

## 快速参考

设计 skill 时，问自己：

1. **这是什么类型？**（纪律 vs. 指导 vs. 参考）
2. **我想改变什么行为？**
3. **哪些原则适用？**（纪律通常是权威 + 承诺）
4. **我是否组合了太多？**（不要使用全部七个）
5. **这是道德的吗？**（服务于用户的真实利益？）
