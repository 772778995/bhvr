# 读书工作台-快速读书并发隔离与标题设计

**当前状态：** 已完成

## 设计结论

允许用户在自动研究运行期间点击 `快速读书`，但 `快速读书` 生成链路必须与自动研究对话线程隔离，不能继续复用“此前对话内容”语义；同时，`快速读书` 按钮在已有总结时恢复为 `快速读书`，不再显示 `重新生成`，而书籍总结标题优先使用模型输出的首个 Markdown 标题，而不是兜底成泛化的“研究报告”。

## 关键决策

1. `快速读书` 继续复用 `POST /api/notebooks/:id/report/generate`，但当 `presetId === builtin-quick-read` 时，服务端 prompt 改为仅基于书籍来源/文档，不再要求综合“此前对话内容”。
2. 自动研究与快速读书允许并发发起；隔离依赖 prompt/线程边界，而不是前端互斥禁用。
3. `快速读书` 按钮文案只区分 `整理中...` 与 `快速读书`，不再出现 `重新生成`。
4. 书籍总结列表中每一项标题优先使用生成结果 Markdown 的第一个 heading；只有没有 heading 时才回退到默认标题。

## 成功标准

- 自动研究运行时可点击 `快速读书`
- 新生成的书籍总结不再明显混入自动研究问答语气或中间追问上下文
- `快速读书` 完成后按钮回到 `快速读书`
- 书籍总结列表标题优先展示模型生成标题，例如“深度参与……”这类自然标题

## 落地状态

- 已直接落地到主工作区 `book-app`
- 无需额外独立 worktree，旧的并行开发 worktree 可在确认无保留价值后回收
