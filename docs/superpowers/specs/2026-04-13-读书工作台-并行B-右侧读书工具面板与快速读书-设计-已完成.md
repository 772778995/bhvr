# 读书工作台并行B-右侧读书工具面板与快速读书设计

**当前状态：** 已完成

## 设计结论

`/book/:id` 右侧不应该继续沿用 `StudioPanel` 的产物工厂语义，而要收缩成读书工具面板：保留自动课题研究，接入快速读书，给快速找书留入口，但去掉原产物九宫格和生成研究报告区域。

## 依赖关系

- 依赖 `前置A-路由入口与共享壳层`
- 可与 `并行A-单书来源面板` 并行开发
- `快速找书` 只保留入口，其独立逻辑在 `独立并行-快速找书`

## 目标

1. 新建 book 专用右侧面板
2. 保留自动课题研究开关与状态展示
3. 接入快速读书能力
4. 预留快速找书入口
5. 删除原产物九宫格与总结按钮

## 组件建议

- `client/src/components/book-workbench/BookActionsPanel.vue`

由 `client/src/views/BookWorkbenchView.vue` 负责接线。

## 保留能力

- 自动课题研究
  - 继续沿用现有研究状态、SSE 和启停逻辑

- 快速读书
  - 复用现有能力
  - 但入口文案和反馈应改成读书场景术语

## 预留能力

- 快速找书
  - 只提供明确按钮或操作入口
  - 不在本任务实现外部书目检索逻辑

## 删除能力

- NotebookLM 原生产物九宫格
- 研究报告总结按钮
- preset 选择与管理弹窗
- 与 audio artifact 等产物相关的配置入口

## 推荐文件边界

- `client/src/components/book-workbench/BookActionsPanel.vue`
- `client/src/views/BookWorkbenchView.vue`
- 若快速读书已有独立 API / helper，则只接最小必要文件

## 验证

1. 右侧不再出现原 Studio 产物区
2. 自动课题研究仍可正常启停
3. 快速读书入口可触发既有能力
4. 快速找书入口可展示为占位或待接入态

## 自审结果

- 已明确右侧面板边界，避免与左侧来源面板和快速找书主逻辑互相污染
