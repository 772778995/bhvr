## 设计结论

中栏 `对话` 面板的消息布局继续沿用当前纸页式聊天方向，但要回到和 `快速找书` 一致的行级对齐模型：用户消息通过独立行容器右对齐，气泡内部文本保持左对齐；助手消息增加最大宽度限制，避免正文横向铺满整栏；输入区底部操作条改为 `items-end`，让发送按钮和双行输入框底边对齐。

## 背景判断

- 这轮聊天样式收口后，`ChatPanel.vue` 只把颜色和输入框皮肤对齐了，但消息行布局没有完全复用 `BookFinderPanel.vue` 的稳定结构。
- 当前 `ChatPanel.vue` 仍然依赖 `ml-auto` 去推用户气泡，这种写法和 `inline-block` 混在一起，表现不稳定，用户肉眼已经看出来歪了，说明不是“吹毛求疵”，是确实没收干净。
- 助手气泡缺少最大宽度约束，长回答会贴着右边一路摊开，阅读节奏直接塌掉，和当前书页式版心完全不搭。
- 输入区按钮容器还在 `items-center`，两行 `textarea` 旁边的发送按钮自然会悬在中线，像是没排版完就上线了。

## 范围边界

本次实现：

- 修正 `ChatPanel.vue` 的用户消息行对齐方式
- 为共享聊天 helper 增加用户文本左对齐与助手最大宽度限制
- 让中栏 `对话` 的发送按钮和输入框底边对齐
- 补测试覆盖，防止样式再次回退

本次不实现：

- 不重做聊天配色和气泡视觉语言
- 不改消息发送逻辑、滚动逻辑和 Markdown 渲染逻辑
- 不改 `快速找书` 的交互语义
