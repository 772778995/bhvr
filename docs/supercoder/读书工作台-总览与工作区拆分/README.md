# 读书工作台总览与工作区拆分设计

**当前状态：** 已完成

## 设计结论

读书工作台的设计已经稳定，但实现边界明显横跨多个独立子任务。继续把所有内容塞在一份总文档里，只会让多人并行开发时互相改同一份规格，最后文档冲突比代码冲突还多。这份文档现在只保留总览、依赖关系和 worktree 拆分规则，具体实现一律下沉到拆分后的子文档。

## 文档拆分规则

为了让文件名直接体现依赖关系，拆分后的文档统一使用以下关键词：

- `前置A`、`前置B`：后续任务的前置，但彼此可并行
- `并行A`、`并行B`：在前置完成后可并行开发
- `收口A`：依赖前面多个并行任务完成后的联调与异常处理
- `独立并行`：不阻塞主线，可单独长期推进
