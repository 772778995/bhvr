## 设计结论

`快速找书` 标签页必须只显示两类内容：固定欢迎气泡，以及快速找书链路自身产生的问答消息；研究历史消息不能混入其中。标签按钮里的“当前标签”辅助文本也不该被用户肉眼看到，应该保持纯装饰隐藏。

## 背景判断

- 现有 `BookFinderPanel` 直接渲染了整份 `messages`
- 现有快速找书消息和研究历史消息共用 `chat_messages` 表，且都标记为 `manual`
- 因此前端若不显式过滤，快速找书页天然会串入普通研究问答历史

## 范围边界

本次只修三件事：

- 去掉标签后面肉眼可见的“当前标签”文本
- 把欢迎文案收成首条 assistant 气泡消息，而不是页眉说明块
- 在快速找书页中过滤消息，只显示欢迎气泡与快速找书自身消息

本次不做：

- 不调整后端消息表结构
- 不回填旧历史数据的来源标记
- 不改动普通研究历史页的展示逻辑

## 推荐方案

### 1. 标签辅助文本

- 保持 `getBookCenterTabIndicatorClass()` 返回 `sr-only`
- 模板中把该 span 标记为 `aria-hidden="true"`
- span 内容置空，避免任何可见冗余文本

### 2. 欢迎气泡

- 将当前 `introTitle + introDescription` 组合为一个固定 assistant 气泡
- 这条欢迎消息不落库，只在快速找书视图层拼接显示
- 欢迎气泡文案固定为：
  - `您好，我是锐读，请输入您要找的书籍内容或类别`
  - `我会先用大模型压缩检索意图，再只从公开书目数据源里整理可核验的候选结果。未核验的平台信息不会硬编。`

### 3. 消息隔离

- 快速找书页只展示：
  - 固定欢迎气泡
  - `id` 以 `book-finder-user:` 开头的乐观用户消息
  - assistant 消息中以 `# 快速找书结果` 开头的结果消息
- 研究历史页继续展示全量历史消息

- 标签按钮后不再出现“当前标签”字样
- 快速找书页打开时首条消息是固定欢迎气泡
- 快速找书页不再显示“课题研究历史”的旧消息
- 发送一次快速找书查询后，只在该页看到欢迎气泡、用户查询和对应找书结果

- 已明确 bug 根因是前端错误复用全量消息，而非标签组件本身

- 已明确欢迎消息属于视图虚拟消息，不写入数据库
